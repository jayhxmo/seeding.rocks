What is this?

Seeding.rocks enables people to access websites as if they were torrents with no effort by the client. Instead of directly sending clients data the main server will redirect people to peers who will give them the data. This enables servers to run with minimal bandwidth. This allows static sites to circumvent the FCC's "slow lane".

Where?

Its used on this page, here, right now!

Why?

Since sites only need to send clients the addresses of their peers bandwidth is kept to a minimum and throttling the servers connection has little to no effect. This model is shown to be resilient in some torrenting solutions that continue to exist despite repeated efforts to stop them. This is also similar in some ways to the "deepnet" with its distributed websites except all client side software is in-browser javascript making its use simple.
